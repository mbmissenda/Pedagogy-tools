# Pedagogy-tools
Two Ways to Change a System
A dual-lens metacognitive scaffold for graduate students designing evidence-informed integrative-health projects inside institutions that weren't built for them. Students hold their project up to two contrasting frameworks — Intervention Mapping and institutional intrapreneurship — and answer the questions each one would ask. A companion diagnostic invites them to learn from a project (theirs or someone else's) that ran out of fuel.

What it is
A single self-contained HTML page. No build step, no server, no JavaScript dependencies beyond Google Fonts. Open the file in any modern browser and the tool works. All student work autosaves to the browser; nothing is sent anywhere.

The tool is designed for asynchronous online graduate learners in integrative health, and is built around three pieces:

-	The Blueprint — a planning lens drawn from Intervention Mapping. Asks the student to specify the problem, the determinants, the change objectives, the theory-based methods, the adoption pathway, and the evaluation.
-	The Build — a planning lens drawn from institutional intrapreneurship. Asks the student to name their free space, field-level status, jurisdiction over resources, trading zones, pipeline of allies, and first small experiment.
-	Why it Stalled — a diagnostic companion (not a third way to change a system). Asks the student to pull the thread on a project that ran out of fuel.

Students also name their evidence base up front — whether they're still seeking the literature, building from an appraisal already in hand, or somewhere in between — so the lenses stay anchored to the work behind the project.
Pedagogical design
This is a metacognitive scaffold, not a worksheet. The tool asks the questions the frameworks would ask; the thinking stays with the student. Specifically:

-	Each lens opens with a short framework primer — open by default on first visit, collapsible thereafter — so a novice has the schema before being asked to use it.
-	The first prompt of each lens offers optional sentence stems to lower activation energy without supplying answers.
-	Each prompt has a "your check-in" rubric of yes/no items that mirror whether the student has done the thinking step (e.g., "I distinguished behaviors from the problem itself"). It never grades whether they got it right.
-	A live progress indicator shows how many prompts in each lens have meaningful content.
-	Work autosaves locally and can be downloaded as a text file, copied to clipboard, or printed as PDF for advisor review.
Theoretical foundations
The Blueprint is drawn from Intervention Mapping:

Fernandez, M. E., Ten Hoor, G. A., van Lieshout, S., Rodriguez, S. A., Beidas, R. S., Parcel, G., Ruiter, R. A. C., Markham, C. M., & Kok, G. (2019). Implementation Mapping: Using Intervention Mapping to develop implementation strategies. Frontiers in Public Health, 7, 209.

The Build is drawn from institutional intrapreneurship:

Heinze, K. L., & Weber, K. (2016). Toward organizational pluralism: Institutional intrapreneurship in integrative medicine. Organization Science, 27(1), 157–172.

The optional reframing construct inside The Build draws on:

Reay, T., Goodrick, E., Waldorff, S. B., & Casebeer, A. (2017). Getting leopards to change their spots: Co-creating a new professional role identity. Academy of Management Journal, 60(3), 1043–1070.

Why it Stalled draws on Heinze & Weber's nursing-faculty cautionary case — an integrative-health nursing program that had grant funding but lacked free space and field-level status, and dissolved when the funding cycle ended.
Using it in teaching
The tool is meant to support — not replace — substantive engagement with the source papers. A few patterns that work:

-	As a pre-class scaffold. Assign the Fernandez and Heinze & Weber papers, then ask students to complete one lens on a project they're considering. Bring the exported text to the synchronous session or office hours.
-	As a comparative reflection. Ask students to complete both lenses on the same project and write a short reflection on where the two views agreed and where they pushed against each other.
-	As a diagnostic on a case. Use the Why it Stalled tab on a published case study or a project a colleague is willing to share, before students apply the two ways to their own work.
-	As a longitudinal artifact. Students return to the tool across the semester as their project evolves. The autosave persists between sessions on the same device; the export lets them carry it forward.

The tool deliberately does not grade or score student responses. Use it alongside the assessments your course already has.
Accessibility
-	Semantic HTML throughout, with ARIA roles on the tabbed lens interface and live regions on the project and evidence echoes.
-	Keyboard navigation: arrow keys move between lenses, Home/End jump to first/last.
-	Visible focus indicators on all interactive elements.
-	Sufficient contrast on text and on the lens-specific accent colors.
-	Print stylesheet reveals all panels for advisor review.

If you find an accessibility gap, please open an issue or contact the author.
Browser support
Designed for current versions of Chrome, Firefox, Safari, and Edge. The tool uses localStorage for autosave; if a student blocks storage, the tool still works in-session but cannot persist work between sessions.
